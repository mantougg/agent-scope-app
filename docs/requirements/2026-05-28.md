# 2026-05-28 需求清单

## R1 — 提交前对话修改 Todo（纯对话驱动）

在 `submit_to_frontend` 真正下发之前，用户能够通过对话让 LLM 调工具改动 Todo 池里的应用 / 模块 / 模型 / 字段数据。

**当前已支持**（Day 4 / Day 5 落地）：
- `update_module(moduleId, newModuleName, newModuleDesc)` — 改模块中文名 / 描述
- `update_model(modelName, appendFieldsJson)` — 给数据模型追加字段

**本次需要补齐的工具**：
- `update_app(newName, newLabel)` — 改应用英文标识 / 中文显示名（`AppSpec` 实际字段是 `name` / `label` / `type`，无 desc；type 分类码不允许改。一次需求会话里通常只有一条 `CREATE_APP` 待办，无需 id 定位，工具内部找唯一一条即可）
- `delete_module(moduleId)` — 删除一个 `CREATE_MODULE` 待办（仅 PENDING）
- `delete_model(modelName)` — 删除一个 `CREATE_MODEL` 待办（仅 PENDING）
- `update_field(modelName, fieldName, newDataType, newComment)` — 改某模型的顶层字段 dataType（long/int/double/string/boolean/date/array）或 comment（中文描述）
- `delete_field(modelName, fieldName)` — 删除某模型的顶层字段

**范围限定**：
- subs 嵌套字段（FieldSpec 的递归子字段）本次不支持编辑，工具显式返回错误提示
- 已经流转到 `RUNNING` / `SUCCESS` / `FAILED` 终态的 Todo 不可改、不可删（沿用 TodoManager 已有约束）

**验收点**：用户在对话中说"把员工模型的 phone 字段类型改成 STRING" / "把第二个模块删掉" / "应用名改成 X" 等指令，LLM 能调对应工具并让看板的 STATE_DELTA 即时反映改动。

## R2 — Chat 助手回复用 markstream-vue 流式渲染 Markdown

把 `ChatPane.vue` 内 assistant 消息的纯文本拼接（`{{ m.text }}`）改成 [`markstream-vue`](https://www.npmjs.com/package/markstream-vue) 的 `<MarkdownRender>` 流式渲染。

**集成范围**：
- 只接 `markstream-vue` 主包 + 内置 Shiki 代码高亮，**不引** Monaco / Mermaid / KaTeX 子模块
- 仅 assistant 气泡使用 markdown 渲染；user 气泡保持纯文本（避免 XSS 路径）
- 流式打字机光标（▍）继续保留，但放到 markdown 渲染区外侧

**验收点**：LLM 回复包含列表 / 代码块 / 加粗等 markdown 语法时能正确渲染；流式过程中不抖动、不重排。

## R3 — HITL 确认按钮内联到 chat 流（取代独立弹窗）

把现有 `HitlConfirmModal.vue` 删除，HITL 确认动作改成 chat 流里的一条助手卡片消息。

**交互行为**：
- 当 `submit_to_frontend` 触发 `ToolSuspendException` 时，前端在 chat 流里 push 一条 `kind='hitl-card'` 的助手消息，列出待提交的 Todo + "确认下发 / 取消"两个按钮
- 点"确认下发" → push `role:'tool' content='USER_CONFIRMED'` 续跑 → LLM 走 `submit_to_frontend(confirmed=true)` 真发
- 点"取消" → push `role:'tool' content='USER_REJECTED'` 续跑 → LLM 按工具描述指引调 **新工具 `cancel_submission`** → TodoManager.clear() 清空所有待办 → 看板 STATE_DELTA(replace /todos []) 同步清空
- 按钮按下后卡片切到 disabled 灰显态，并显示"已确认下发"或"已取消（清空待办）"文字反馈，保留在 chat 历史里可追溯

**验收点**：HITL 触发时不弹任何 modal，按钮就贴在对话流里；取消后看板清空、对话继续；用户可以滚动看到历史中的 HITL 卡片以及最终决定。
